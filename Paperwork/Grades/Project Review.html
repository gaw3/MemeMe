Reviewer Comments

To eliminate verbosity, improve code clarity, and reduce typing time, it's generally recommended to only use the self keyword when required by the compiler. When it's not required {e.g. code is outside of a closure, no naming conflicts with global and local variables) it's safe to omit the keyword. There are several instances throughout the project where the keyword is used when not required and a few examples have been highlighted in the comments. But by following these general guidelines, it should be fairly easy to track down any other unnecessary usage.
